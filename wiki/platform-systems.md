# 中台系统全景图

> 我统筹管理的 8 个中台系统，涵盖订单、支付、会员、促销、商品、菜单、推荐、活动管理。

## 系统一览

| 系统 | 一句话定位 | 关键词 |
|------|-----------|--------|
| Order | 全渠道统一中台订单系统，核心中的核心 | 订单、全渠道、App/AIO/3PO/团餐 |
| Payment | 支付&退款能力 + 开票 | 支付、退款、第三方支付渠道 |
| CRM | 会员&积分管理 | 会员信息、积分 |
| Promotion | 促销卡券（优惠计算 + 卡券管理） | 促销活动、优惠券、权益卡 |
| PCM | 商品主数据管理 | 商品、价格、售卖时间、多平台 |
| Menu | 菜单布局与发布 | 菜单模版、菜单生成、上下架 |
| Suggestion | 个性化商品推荐 | 推荐、转化率、多点位 |
| Campaign | 活动配置需求收集与工单管理 | 活动创建、配置工单、运营协同 |

## 系统详情

### Order（订单系统）⭐ 核心

全渠道集中统一的中台订单系统，支持 App、AIO、3PO、团餐等全渠道订单业务；集成到店取餐、麦乐送、麦咖啡、DT、取餐柜等各种业务类型的订单。中台核心系统。

### Payment（支付系统）

提供完整的支付&退款能力，建立起订单与第三方支付渠道之间的完整支付能力，另外还包括开票功能。

### CRM（会员&积分系统）

会员信息、会员积分等相关信息的管理功能。

### Promotion（促销卡券系统）

两大模块：
- **促销**：优惠计算、促销活动预计算功能、促销活动信息等
- **卡券**：用户可用优惠券、权益卡、优惠券创建和管理等

### PCM（商品主数据系统）

商品管理（单品、套餐、商品组等），商品基本信息维护（名称、属性、编码、图片、价格、售卖时间等），为 APP/小程序/New Web/3PO/CNSOK 等平台提供 by 门店/daypart/日期类型/售卖时间等维度的商品和菜单数据配置、生成能力。

### Menu（菜单系统）

管理菜单布局和售卖商品范围，按陈列布局将 PCM 系统中的商品组织到一起。主要功能：菜单模版配置、菜单规则配置、菜单生成、菜单发布、上下架逻辑。

### Suggestion（推荐系统）

多点位推荐满足用户架构需求，个性化商品推荐链路，增加商品曝光同时提升订单转化率。

### Campaign（活动管理系统）

为大中台系统及前端配置系统提供线上收集配置需求服务。业务人员在 Campaign 中创建活动，选择卡券、资源位、H5 页面等类别的配置需求（未来还有商品、菜单、促销），产生配置工单分派给各运营团队。运营人员收到工单后到中台/前端系统完成配置，再回 Campaign 更新工单状态。

## 系统间依赖关系（上下游调用链）

> 待日常使用过程中随时补充

初步已知依赖（方向：上游 -> 下游）：

- PCM -> Menu（商品数据 -> 菜单组织）
- Promotion -> Order（优惠计算 -> 订单价格）
- Payment -> Order（支付能力 -> 订单完成）
- CRM -> Promotion（会员信息 -> 券发放/核销）
- Campaign -> Promotion / PCM / Menu（活动工单 -> 各系统配置执行）
- Suggestion -> PCM + Order（商品数据 + 订单数据 -> 推荐模型）
