# RCA 复盘方法论

> 我的标准根因分析与复盘工作流，所有复盘/RCA 任务的统一入口。

## 一句话方法论

> **4 要素准备 → 5-Why 纵深挖根 → AI 3 层追问外推 → 8 章节结构化输出**

## 何时调用此方法论

| 场景 | 是否适用 |
|------|---------|
| 线上故障复盘 | ✅ |
| 部署演练复盘（BCP/容灾/混沌） | ✅ |
| 测试发现的成组缺陷复盘 | ✅ |
| 个人项目踩坑复盘 | ✅ |
| 单点 bug 调试 | ❌ 用 [[ai-debug-kafka-consumer-switch-race-condition]] 的"现象+上下文+假设"模式 |
| 测试报告（关注现状不挖根因） | ❌ 用 [[ai-test-report-template]] |

## 三层方法论叠加

### 第一层：4 要素准备（缺一不可）

| 要素 | 关键问题 | 缺失后果 |
|------|---------|---------|
| 现象描述 | 什么"不对"了？ | AI 没法定位问题 |
| 时间线 | 什么时候开始/发现/修复？ | 无法判断因果顺序 |
| 影响范围 | 谁/什么受影响？严重度多少？ | 无法划定优先级 |
| 已知线索 | 已查到的代码/日志/配置 | AI 会瞎编填补 |

**铁律**：任一要素缺失，先要求用户补齐再开始 5-Why，不能凭空推。

### 第二层：5-Why 纵向深挖

**判断挖到位的信号**：

- ✅ 最后一层回答的是"假设 / 规范 / 认知"层面
- ✅ 修复方案不再是"改一行代码"，而是"改流程/补规范"
- ❌ 如果最后一层还在说"因为没注意到 XXX"，那是借口不是根因

**Prompt 模板**：

```
请基于以下信息，对问题 [问题描述] 做 5-Why 根因分析。

【现象】{现象描述}
【时间线】{时间线}
【影响范围】{影响范围}
【已知线索】{代码/日志/配置线索}

要求：
1. 每一层 Why 都基于上一层的"果"，不要跳跃
2. 至少深挖 5 层
3. 当回答触及"架构假设/流程缺陷/团队认知"层面时，标记为【根因】
4. 不要止步于"这是个 bug"，要回答"为什么会出现这个 bug"
```

### 第三层：AI 3 层追问（防止单点修复）

5-Why 只能挖深度，挖不到广度。必须追加三个维度：

| 追问维度 | 句式 | 产出 |
|---------|------|------|
| **检测维度** | "为什么这个缺陷没在 UT/IT/CodeReview/灰度阶段被拦截？" | 质量保障缺口清单 |
| **流程维度** | "团队的什么规范/流程/工具如果存在，就能避免？" | 可执行的改进措施 |
| **同类风险** | "代码库里还有哪些组件可能存在同源问题？" | 排查清单（防复发） |

> 💡 同类风险追问是 RCA 与事后报告的核心差异——前者会主动外推。

## 报告结构（8 章节硬性要求）

```
1. 复盘目的（回答的核心问题，不是流水账）
2. 事件时间线（带"观察"，不只是时刻表）
3. 5-Why 深度分析（每个问题或问题组一段）
4. 多层根因归纳（表层/中层/深层）
5. 拦截环节复盘（应有拦截手段 vs 实际状态）
6. 同类风险外推（≥ 4 类风险类型）
7. 改进措施（P0/P1/P2 + 责任方 + 截止 + 验收标准）
8. 经验沉淀（"带走 3 件事"洞察，不重复前文）
```

## 一键 Prompt（生成完整复盘）

```
你是一名资深的质量分析专家，请基于以下材料生成完整 RCA 复盘报告。

【背景】
{项目背景、为什么要做这个复盘}

【问题清单】
{逐个列出问题，包括现象/时间线/影响/已知线索/当前状态}

【输出要求】
1. 包含 8 章节：复盘目的/时间线/5-Why/多层根因/拦截复盘/同类风险/改进措施/经验沉淀
2. 5-Why 必须挖到"架构假设/流程缺陷/团队认知"层面才停
3. 改进措施带 P 级、负责方、截止时间、验收标准
4. 同类风险至少 4 类，每类说明排查范围和严重度
5. 经验沉淀给 3 条"带得走"洞察，不重复前文

格式：Markdown，含表格，便于团队评审。
```

## 常见陷阱（先记住，再避坑）

| 陷阱 | 表现 | 避免方法 |
|------|------|---------|
| 5-Why 走形 | 5 层都在说"代码层面"，没触及架构/流程 | 强制最后一层回答"假设/规范/认知" |
| 根因泛化 | 写成"测试不够""人员不足"等虚词 | 每个根因必须可对应 ≥1 条改进措施 |
| 只修不防 | 改进措施只针对当前问题，无外推 | 必须包含"同类风险排查"措施 |
| 责任甩锅 | 归结为"某人没注意" | 系统问题不归个人，问"为什么流程允许个人疏漏导致事故" |
| AI 瞎编 | AI 把不存在的代码/配置说得很真 | 4 要素必须真实，关键结论人工核对 |
