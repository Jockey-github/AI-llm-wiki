# 实战案例：Zero-shot vs Few-shot 生成测试用例效果对比

> 日期：2026-04-28 | AI Mastery Day 2 | M1 Prompt 工程

## 背景

用同一个真实业务需求（美团套餐可选组同步逻辑），分别用 Zero-shot 和 Few-shot 结构化 Prompt 让 AI 生成测试用例，对比输出质量差异。

## 需求原文

美团套餐可选组在 X选X（2选2，3选3）的情况下，不再转成固定组，按可选组同步。

## 实验一：Zero-shot（裸 Prompt）

**输入：**

```
帮我根据以下需求生成一份完整的测试用例集
美团套餐可选组在X选X（2选2，3选3）的情况下，不再转成固定组，按可选组同步
```

**输出特征：**
- 生成 10 条用例
- 基本围绕"X选X 数量变化"单一维度
- 预期结果模糊（如"可选组保持为可选组同步"）
- 需求理解靠猜，不确定同步方向和业务上下文
- 缺少用户选择行为、加价、售罄等真实场景

## 实验二：Few-shot 结构化 Prompt

**输入：**

```markdown
帮我根据以下需求生成一份完整的测试用例集

# 需求内容：
 - 美团套餐可选组在X选X（2选2，3选3）的情况下，不再转成固定组，按可选组同步

# 当前现状：
 - 美团套餐在推送给第三方的时候，如果套餐中存在可选组，则会把可选组中的所有商品固定下来，而不是给用户进行选择
 - 举个例子：套餐A中存在一个3选3的饮料可选组，可选组中有三个商品可乐、雪碧、咖啡。当前处理逻辑是转成固定组之后，直接给到用户可乐、雪碧、咖啡各一个。

# 想要达到的效果：
 - 还是基于上面当前现状中的例子：优化完成之后，不会再是固定组，而是会给到用户进行选择。用户可以选择3杯可乐，也可以选择1杯可乐，2杯雪碧等。
```

**输出特征：**
- 生成 20 条用例，按 4 个维度分组（正向功能 / 套餐组合 / 边界异常 / 数据一致性）
- 深入到用户选择行为（重复选、混合选、数量校验）
- 覆盖加价逻辑、商品售罄、存量数据迁移等真实场景
- 预期结果具体可验证（如"总价=套餐价+9元"）
- 用真实商品名称描述，可读性强

## 效果对比

| 维度 | Zero-shot | Few-shot 结构化 |
|------|-----------|----------------|
| 用例数量 | 10 条 | 20 条 |
| 需求理解 | 靠猜，不确定同步方向 | 精准理解变更前后行为 |
| 覆盖维度 | 仅同步维度 | 功能 + 组合 + 边界 + 数据 |
| 业务贴合度 | 泛泛而谈 | 贴合真实业务场景 |
| 预期结果 | 模糊笼统 | 具体可验证 |
| 可直接执行 | 需大量补充 | 基本可直接用 |

## 核心结论

1. **上下文是关键**：给 AI 当前现状 + 具体示例 + 期望效果，比单纯描述需求有效 10 倍
2. **Few-shot 的本质**：不是"给例子"，而是帮 AI 建立正确的心智模型
3. **结构化的价值**：Markdown 分块让 AI 能区分"是什么 / 现在怎样 / 要变成怎样"，避免信息混淆

## 可复用的测试用例生成 Prompt 模板

```markdown
帮我根据以下需求生成一份完整的测试用例集

# 需求内容：
 - {一句话描述需求变更}

# 当前现状：
 - {描述当前系统行为}
 - 举个例子：{用具体数据说明当前行为}

# 想要达到的效果：
 - {基于同一个例子，说明变更后的期望行为}

# 输出要求：
 - 按维度分组：正向功能 / 组合场景 / 边界异常 / 数据一致性
 - 表格格式：编号 / 场景 / 前置条件 / 操作步骤 / 预期结果 / 优先级
 - 预期结果要具体可验证
```
