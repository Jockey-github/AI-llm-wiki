# AI 辅助 SQL 数据分析实战

> Day 8 微项目 | 2026-06-12

## 场景

查询 `t_pcm_category` 表中每个门店下的商品分类数量统计。

## 给 AI 的输入

- **表名**：`t_pcm_category`
- **关键字段**：`store_code`（门店编码）、`category_code`（分类编码）
- **分析目标**：统计每个门店下有多少条商品分类记录

## AI 生成的 SQL

```sql
SELECT store_code, COUNT(category_code) AS category_count 
FROM t_pcm_category 
GROUP BY store_code 
ORDER BY category_count DESC;
```

## 执行结果

| store_code | category_count |
|------------|---------------|
| 1450997    | 618           |
| 1990999    | 198           |

## 关键经验

### 给 AI 写 SQL 必须提供的信息

1. **表结构**（DDL 或关键字段说明）— 没有表结构 AI 只能瞎猜
2. **分析目标**（用自然语言描述你想知道什么）
3. **数据库类型**（MySQL / TiDB / PG，影响语法差异）
4. **约束条件**（时间范围、过滤条件、排序要求）

### AI 辅助 SQL 的最佳实践

| 步骤 | 说明 |
|------|------|
| 1. 明确问题 | 把业务问题转化为数据问题 |
| 2. 提供表结构 | DDL 或字段说明，越详细越好 |
| 3. AI 生成 SQ 描述需求，让 AI 生成初版 |
| 4. 本地验证 | 先在测试环境跑一遍确认结果 |
| 5. 优化迭代 | 结果不对或性能差时，给 AI 反馈让它改进 |

### 适合让 AI 写的 SQL 类型

- 复杂 JOIN（多表关联）
- 窗口函数（排名、累计、环比）
- 子查询嵌套
- 数据透视 / 交叉统计
- 执行计划分析 & 索引建议
