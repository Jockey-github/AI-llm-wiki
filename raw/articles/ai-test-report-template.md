# AI 辅助生成测试报告 · 工作流模板

## 适用场景

- 测试阶段结束后需要输出正式测试报告
- 演练/验证活动后需要结构化总结
- 任何需要从散乱记录中提炼结论的报告场景

## 三步迭代法

### Step 1：骨架确认（2min）

明确以下要素后再动手：

| 要素 | 选项 |
|------|------|
| 受众 | 项目组内部 / 管理层 / 两者兼顾 |
| 深度 | 完整版 / 精简版 / 侧重问题分析 |
| 数据源 | 演练文档 / Jira / 测试平台 / 日志 |

### Step 2：初稿生成（3-5min）

给 AI 的 Prompt 结构：

```
基于以下文档/数据，生成一份测试报告，要求：
- 受众：{受众}
- 包含模块：测试概述、测试范围、执行统计、缺陷分析、风险评估、结论建议
- 统计数据用表格呈现
- 缺陷按根因分类
- 风险评估标注等级和影响范围
- 结论给出明确的通过/不通过判定

{粘贴数据源内容}
```

### Step 3：迭代润色（5min）

逐段检查：
- [ ] 数据是否准确（通过率、问题数等）
- [ ] 根因分类是否合理
- [ ] 风险等级是否符合实际判断
- [ ] 改进建议是否可执行（有责任方+时间）
- [ ] 措辞是否匹配受众

## 报告标准结构

```
1. 测试概述（背景/目标/时间/人员）
2. 测试范围（维度 + 检查项数）
3. 测试执行统计（总数/通过/失败/阻塞/通过率 + 各维度明细）
4. 缺陷分析（问题汇总表 + 根因分类 + 关键发现）
5. 风险评估（遗留风险 + 流程风险）
6. 结论与建议（结论判定 + 改进建议 + 准入条件）
7. 附录（问题明细/技术细节）
```

## 效率对比

| 环节 | 手写耗时 | AI辅助耗时 | AI擅长点 |
|------|---------|-----------|---------|
| 数据提取统计 | 30min | 1min | 从散乱文档中归纳数字 |
| 结构搭建 | 15min | 0min | 模板即用 |
| 内容填充 | 45min | 3min | 扩写和格式化 |
| 分析归类 | 20min | 1min | 根因分类、风险矩阵 |
| 人工审校 | — | 5min | 需要人判断准确性 |
| **总计** | **~2h** | **~10min** | |

## 注意事项

- AI 生成的数据统计需要人工核对，尤其是通过率计算
- 风险等级和结论判定需要人工确认，AI 倾向保守
- 根因分类可能过于笼统，需要结合实际技术背景调整
- 改进建议的时间节点和责任方需要人工补充
