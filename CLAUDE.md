# AI-llm-wiki Schema

这是一个关于「我」的个人知识库，由 LLM 增量构建和维护。记录关于我的一切：决策、习惯、偏好、成长轨迹。

## 架构

```
AI-llm-wiki/
├── CLAUDE.md          # 本文件 - schema，定义 wiki 的结构和维护规则
├── raw/               # 原始素材（不可修改）
│   ├── articles/      # 剪藏的文章、笔记
│   ├── notes/         # 日记、随手记、会议记录
│   └── assets/        # 图片等附件
├── wiki/              # LLM 生成和维护的知识页面
│   ├── index.md       # 内容索引（按分类组织）
│   ├── log.md         # 操作日志（按时间线）
│   └── *.md           # 各主题页面
└── llm-wiki.md        # 参考：Karpathy 的原始方案说明
```

## Wiki 页面分类

| 分类 | 说明 | 示例页面 |
|------|------|---------|
| 身份 | 个人基本信息、角色定位 | me.md |
| 工作 | 职责、团队、日常 | work.md, team.md |
| 技术 | 技术栈、能力水平、学习路径 | tech-stack.md, learning.md |
| 项目 | 当前和历史项目 | projects.md |
| 偏好 | 工具、习惯、沟通风格 | preferences.md |
| 目标 | 短期/长期目标、计划 | goals.md |
| 决策 | 重要决策及其推理过程 | decisions.md |
| 经验 | 踩坑记录、复盘总结 | lessons.md |
| 健康 | 身体、运动、饮食 | health.md |
| 兴趣 | 爱好、阅读、生活 | interests.md |

## 页面格式

每个 wiki 页面使用以下格式：

```markdown
---
title: 页面标题
category: 分类
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [相关原始素材路径]
---

# 页面标题

正文内容...

## 相关页面

- [[相关页面1]]
- [[相关页面2]]
```

## 操作流程

### Ingest（摄入素材）

1. 用户将素材放入 raw/ 目录（或直接在对话中告知）
2. LLM 读取素材，提取关键信息
3. 创建或更新相关 wiki 页面
4. 更新 index.md 索引
5. 追加 log.md 日志条目

### Query（查询）

1. 用户提问
2. LLM 读取 index.md 定位相关页面
3. 读取相关页面内容
4. 综合回答，附带引用
5. 如果答案有普遍价值，建议存回 wiki 作为新页面

### Lint（健康检查）

定期检查 wiki 质量：

- 页面间是否存在矛盾信息
- 哪些页面长期未更新（可能已过时）
- 是否有孤立页面（无入链接）
- 是否有被频繁提及但尚无独立页面的概念
- 是否有数据空白可以通过追问用户来补充

## 维护原则

1. **用户不写 wiki，LLM 写**：用户负责提供素材和方向，LLM 负责整理和维护
2. **增量更新**：每次只更新变化的部分，不重写整个页面
3. **交叉引用**：页面之间用 `[[页面名]]` 互相链接，构建知识网络
4. **矛盾标记**：当新信息与已有记录冲突时，明确标注并提请用户确认
5. **时间戳**：所有更新都带时间戳，便于追踪信息的新鲜度
6. **隐私优先**：敏感信息（密码、密钥等）绝不记录在 wiki 中

## 与 Claude Code Memory 的关系

- Claude Code 的 `~/.claude/projects/*/memory/` 存放**会话级**的行为指令和项目进展
- 本 wiki 存放**长期积累**的关于用户的结构化知识
- 两者互补：memory 指导当前行为，wiki 提供深度背景

## 交互约定

- 当用户说"记录一下"或"加到知识库"时，执行 Ingest 流程
- 当用户说"我的知识库里有没有..."或"外脑"时，执行 Query 流程
- 当用户说"检查一下知识库"时，执行 Lint 流程
- 日常对话中发现值得记录的信息时，主动建议是否加入 wiki
